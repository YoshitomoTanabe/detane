# detane 向けエージェント手順

このリポジトリは、TypeScript の CDK アプリです。フロントエンドは React、バックエンドは S3 / Lambda / DynamoDB を前提にしています。

## まず読むもの

- `AGENTS.md`
- `cdk.json`
- `package.json`
- `lib/detane-stack.ts`
- `bin/detane.ts`
- `test/`

## 作業ルール

- 既存の構成と命名に合わせること。
- 変更はできるだけ小さく、局所的にすること。
- 役割の違うコードを混ぜないこと。
- 本体の construct と検証用 harness は分けて扱うこと。
- 仕様が曖昧なときは、最小実装を優先すること。

## CDK について

- `aws-cdk-lib` と `constructs` を使う。
- スタックは、特別な理由がない限り env 非依存で保つ。
- 新しいリソースを追加したら、`aws-cdk-lib/assertions.Template` で検証する。
- S3 / Lambda / DynamoDB のいずれも、存在確認だけでなく重要な設定も確認する。

## React について

- React はこのスタックのフロントエンドとして扱う。
- フロントエンド資産を追加するなら、ビルド出力と配信経路を明示する。
- 静的配信だと決め打ちしない。実装に合わせて扱う。

## バックエンドについて

- Lambda は小さく保ち、イベントソースと IAM 権限を明示する。
- DynamoDB はアクセスパターンに合わせて最小構成で設計する。
- S3 は用途を限定して使う。フロントのホスティングかオブジェクト保存を基本にする。

## AI 向け

- 変更前に、このファイルと関連ファイルを読むこと。
- インフラを変えたら、synth 結果か assertions で確認すること。
- 新しい動作を足すなら、stack と harness を同時に更新すること。
- 推測だけのリファクタは避けること。
